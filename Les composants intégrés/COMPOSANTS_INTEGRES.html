Les composants intégrés

Bootstrap propose des composants pratiques à mettre en oeuvre : barres de navigation, effets typographiques, panneaux, thumbnails... et tout ça sans écrire une seule ligne de code JavaScript. Je verrai aussi alertes et barres de progression qui demandent un peu de JavaScript pour être utilisées...

2.) Une barre de navigation

a./ Principe de base

Bootstrap propose de superbes classes pour créer une barre de navigation. A la base, une barre nécessite juste les classes navbar et navbar-default : < nav class="navbar navbar-default"> Je suis une barre < /nav> Ce code donne la figure suivante : La classe navbar est toute simple : .navbar { position: relative; min-height: 50px; margin-bottom: 20px; border: 1px solid transparent; } Et la classe navbar-default fixe les couleurs du fond et de la bordure : .navbar-default { background-color: #f8f8f8; border-color: #e7e7e7; } Pourquoi 2 classes ? Je verrai plus loin qu'il est possible d'avoir une barre "inversée", avec une couleur de fond différente. Mais une barre vide ne sert pas à grand chose, en général on la garnit d'items. Voici un exemple : < div class="navbar navbar-default"> < ul class="nav navbar-nav"> < li class="active"> < a href="#">Accueil< /a> < /li> < li> < a href="#">Liens< /a> < /> < li> < a href="#">Témoignages< /a> < /li> < li class="disabled"> < a href="#">Préférences< /a> < /li> < /ul> < /div> Les classes nav et navbar-nav doivent être utilisées avec une balise < ul>. Tous les items de navigation apparaîssent alors avec des balises < li>. L'item actif est repéré avec la classe active. Un item désactivé est défini avec la classe disabled. Voici à la figure suivante un aperçu de la barre.

b./ Barre avec un fond noir

Si je préfère une barre avec un fond noir, il suffit de remplacer la classe navbar-default par la classe navbar-inverse : < div class="navbar navbar-inverse"> < ul class="nav navbar-nav"> < li class="active"> <a href="#">Accueil</a> </li> < li> <a href="#">Liens</a> </li> <li> <a href="#">Témoignages</a> </li> <li class="disabled"> <a href="#">Préférences</a> l&t;/li> </ul> </div> On obtient alors le résultat de la figure suivante.

c./ Un titre dans la barre

On peut faire apparaître un titre avec des caractères plus visibles avec la classe navbar-brand : <div class="navbar navbar-default"> <div class="navbar-header"> <a class="navbar-brand" href="#">Mon beau site</a> </div> <ul class="nav navbar-nav"> <li class="active"> <a href="#">Accueil</a> </li> <li> <a href="#">Liens</a> </li> <li> <a href="#">Témoignages</a> </li> <li> <a href="#">Préférences</a> </li> <li> <a href="#">Bibliographie</a> </li> </ul> </div> Illustration : Le titre doit être inclus dans une classe navbar-header pour avoir un affichage correct sur les mobiles.

d./ Fixer la barre

A priori, on veut une barre de navigation dans la partie supérieure d'un site, mais en général sous un bandeau. Avec le code tel qu'on l'a vu jusque-là, la barre se situe dans le flux. Mais on peut fixer la barre en haut de page du navigateur avec la classe navbar-fixed-top : <p>Un paragraphe</p> <div class="navbar navbar-default navbar-fixed-top"> <div class="container"> <div class="navbar-header"> <a class="navbar-brand" href="#">Mon beau site</a> </div> <ul class="nav navbar-nav"> <li class="active"> <a href="#">Accueil</a> </li> <li> <a href="#">Liens</a> </li> <li> <a href="#">Témoignages</a> </li> <li> <a href="#">Préférences</a> </li> <li> <a href="#">Bibliographie</a> </li> </ul> </div> </div> <p>Un autre paragraphe</p> <p>Encore un paragraphe</p> Illustration :

Attention: L'utilisation de la classe navbar-fixed-top impose la présence de la classe container ou container-fluid

La barre prend alors toute la largeur de la fenêtre. Les éléments de la page se déplacent sous la barre fixe en cas de scrolling. Mais où sont passés les 2 premiers paragraphes pourrait-on se demander à juste titre... Eh bien, ils sont sous la barre ! Il suffit de regarder la classe pour comprendre : .navbar-fixed-top, .navbar-fixed-bottom { position: fixed; right: 0; top: 0; z-index: 1030; } La barre est en positinnement fixe en haut de la page du navigateur avec un z-index assez conséquent... Donc si on adopte un positionnement fixe en haut de la page du navigateur, il faut prendre la précaution de ménager suffisamment d'espace pour que les éléments ne soient pas masqués par la barre : <link href="bootstrap/css/bootsrap.css" rel="stylesheet"> <style> body { padding-top: 70px; } </style> Résultat : Il existe aussi la classe navbar-fixed-bottom pour fixer la barre en bas de la page. Mais les cas d'utilisation sont plus anecdotiques. Le principe demeure le même.

e./ Une barre statique

Et si je désire que la barre suive le mouvement et ne soit plus collée au bord de la fenêtre (comme avec la classe navbar-fixed-top), mais au-dessus du contenu, de telle sorte que la barre disparaisse quand je fais défiler, je dois alors utiliser la classe navbar-static-top : <div class="navbar navbar-default navbar-static-top"> WTF : Je ne vois pas pourquoi j'uiliserais cette classe, puisqu'il suffit de ne pas utiliser l'autre classe pour que la barre ne soit pas fixée. La seule chose que la classe navbar-static-top apporte de plus est, apparemment, l'arrondi de la barre qui devient rectangle.

f./ Un formulaire de recherche dans la barre

Pour intégrer un formulaire il suffit d'ajouter une balise <form> avec la classe navbar-form. On a 2 classes pour décaler ce formulaire pour l'un des côtés : navbar-left : pour le fixer à gauche navbar-right : pour le fixer à droite Voici l'intégration pour ma page d'exemple : <nav class="navbar navbar-inverse"> <div class="container-fluid"> <ul class="nav navbar-nav"> <li class="active"> <a href="#">Accueil</a> </li> <li> <a href="#">Liens</a> </li> <li> <a href="#">Témoignages</a> </li> <li> <a href="#">Préférences</a> </li> <li> <a href="#">Bibliographie</a> </li> </ul> <form class="navbar-form navbar-right inline-form"> <div class="form-group"> <input type="search" class="input-sm form-control" placeholder="Recherche"> <button type="submit" class="btn btn-primary btn-sm"><span class="glyphicon glyphicon-ok-sign"></span> Chercher<button> </div> </form> </div> </nav> Illustration : Je retrouve bien mon formulaire décalé à droite. Concernant la classe inline-form... Ce n'est pas une classe de Bootstrap ! Mais une classe qu'on a dû ajouter pour obtenir un comportement correct du formulaire. Voilà ce qui se produit si je réduis la largeur de la fenêtre sans cette classe : J'ai en effet deux soucis : - le contrôle prend la largeur - le bouton est renvoyé en-dessous Il faut donc que je modifie le display du contrôle et que je limite sa largeur. Ce qui est fait par la classe inline-form : .inline-form input { dispay: inline-block; width: 100px; } Maintenant, j'ai un affichage correct que l'on peut voir à la figure suivante : Et pourquoi ne pas utiliser la classe form-inline de Bootstrap ? Cette classe ne fonctionne que si la largeur d'affichage est d'au moins 768px, et c'est justement au-dessous qu'on rencontre le problème !

g./ Bouton et texte dans une barre

Il existe la classe navbar-btn pour intégrer un bouton dans une barre, mais qui doit être utilisée uniquement si le bouton n'appartient pas à un formulaire. Il y a aussi la classe navbar-text pour insérer du texte et la classe navbar-link pour afficher un lien en-dehors du menu (non inclus dans la classe nav). Voici un exemple : <nav class="navbar navbar-default"> <div class="navbar-header"> <a href="#" class="navbar-brand">Un titre</a> </div> <p class="navbar-text">Un simple texte</p> <button type="button" class="btn btn-default navbar-btn">Bouton</button> <p class="navbar-text"><a href="#" class="navbar-link">Lien</a></p> </nav> Ce qui donne :

En résumé

Bootstrap permet de créer une barre de navigation fixe ou mobile qui peut comporter un titre, des liens, des boutons et des formulaires. Bootstrap possède un composant pour la navigation entre pages Web. Pour des effets visuels Bootstrap propose son Jumbotron, des libellés, des badges et des thumbnails pour les images. Les listes groupées peuvent comporter des badges, des liens, des images et être coloréees. Les panneaux permettent d'isoler visuellement une partie de la page et peuvent comporter un titre, ils peuvent intégrer facilement une liste groupée ou un tableau. Le composant media permet de proposer des images, des musiques, des vidéos. Bootstrap permet l'affichage de bandeaux d'alerte colorés et de barres de progression.
